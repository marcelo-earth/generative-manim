# Training Dataset Format

This document describes the JSONL file formats used at each stage of the Generative Manim training pipeline.

## File locations

All intermediate and final datasets live under `training/data/outputs/`:

```
training/data/outputs/
├── raw_prompts.jsonl          # Stage 1: expanded prompts
├── raw_completions.jsonl      # Stage 2: teacher completions
├── validated_completions.jsonl # Stage 3: render-validated completions
├── sft_train.jsonl            # Stage 4: SFT training split
├── sft_val.jsonl              # Stage 4: SFT validation split
├── sft_test.jsonl             # Stage 4: SFT test split
└── test_prompts.jsonl         # Prompts from test split for evaluation
```

## Format by stage

### `raw_prompts.jsonl`

One prompt per line, generated by GPT-4o from seed categories.

```json
{"category": "geometry", "prompt": "Draw a blue equilateral triangle that rotates 360 degrees."}
```

| Field | Type | Description |
|-------|------|-------------|
| `category` | string | One of the 15 prompt categories (e.g. geometry, physics, algebra) |
| `prompt` | string | Natural-language animation description |

### `raw_completions.jsonl`

Teacher-model (GPT-4o) completions for each prompt.

```json
{"category": "geometry", "prompt": "Draw a blue equilateral triangle...", "completion": "from manim import *\n\nclass GenScene(Scene):\n    def construct(self):\n        ...", "model": "gpt-4o"}
```

| Field | Type | Description |
|-------|------|-------------|
| `category` | string | Inherited from the prompt |
| `prompt` | string | Original prompt |
| `completion` | string | Raw LLM output (may include markdown fences) |
| `model` | string | Model used for generation |

### `validated_completions.jsonl`

Only completions that rendered successfully. Code is cleaned (markdown fences stripped, imports normalized).

```json
{"category": "geometry", "prompt": "Draw a blue equilateral triangle...", "code": "from manim import *\n\nclass GenScene(Scene):\n    def construct(self):\n        ...", "animation_count": 2, "render_time": 14.3}
```

| Field | Type | Description |
|-------|------|-------------|
| `category` | string | Prompt category |
| `prompt` | string | Original prompt |
| `code` | string | Cleaned, render-verified Manim code |
| `animation_count` | int | Number of `self.play()` calls detected |
| `render_time` | float | Seconds taken to render |

### `sft_train.jsonl` / `sft_val.jsonl` / `sft_test.jsonl`

Final chat-format dataset ready for SFT training with TRL `SFTTrainer`.

```json
{"messages": [{"role": "system", "content": "You are an assistant that creates animations with Manim..."}, {"role": "user", "content": "Draw a blue equilateral triangle..."}, {"role": "assistant", "content": "from manim import *\n\nclass GenScene(Scene):\n    def construct(self):\n        ..."}]}
```

| Field | Type | Description |
|-------|------|-------------|
| `messages` | array | Three-message conversation: system, user, assistant |
| `messages[0].role` | `"system"` | Always `"system"` |
| `messages[0].content` | string | Manim system prompt from `training/utils/system_prompt.py` |
| `messages[1].role` | `"user"` | Always `"user"` |
| `messages[1].content` | string | The animation prompt |
| `messages[2].role` | `"assistant"` | Always `"assistant"` |
| `messages[2].content` | string | The Manim Python code |

Note: the `source` metadata field is stripped in the final SFT files to keep them clean for training.

### `test_prompts.jsonl`

Prompts extracted from the test split, used by the evaluation harness.

```json
{"prompt": "Draw a blue equilateral triangle that rotates 360 degrees."}
```

## Default split sizes

The `prepare_sft_dataset.py` script splits the validated dataset as follows:

- **Test**: up to 50 examples (capped at 20% of total)
- **Val**: 10% of the remaining examples
- **Train**: the rest

Example with 2000 validated examples: Train=1755, Val=195, Test=50.

## External datasets

Additional datasets merged in during `prepare_sft_dataset.py`:

| Dataset | Source | Format |
|---------|--------|--------|
| `edoh-dataset-format-pep8.jsonl` | `datasets/` directory | `messages` format |
| `physics-01.jsonl` | `datasets/` directory | `messages` format |
| `dataset-2025-02-15T02-45-51-960Z.jsonl` | Repo root | `messages` format |
| `claude_generated.jsonl` | `training/data/outputs/` | `prompt`/`code` format |

All external datasets are deduplicated by prompt text before the train/val/test split.
